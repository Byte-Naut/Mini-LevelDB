# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Build and run

- Configure + build (CMake):
  - `cmake -S . -B build`
  - `cmake --build build`
- Run the built target:
  - `./build/Mini_LevelDB`

## Testing and verification

- There is no unit-test framework wired in this repo yet (no CTest/GoogleTest target).
- Load/concurrency verification is currently done with:
  - `python3 stress_test.py`
- If you need a "single test" style run, reduce `TOTAL_REQUESTS` and/or `CONCURRENCY_LIMIT` in `stress_test.py` and run the same script.

## Important build reality

- `CMakeLists.txt` currently builds `main.cpp` as the executable entrypoint.
- `server/reactor_server.cpp`, `server/epoll_server.cpp`, and `server/blocking_server.cpp` are alternative server experiments and are **not** currently part of the CMake target.
- Before changing protocol or stress behavior, verify which server entrypoint is actually being compiled and executed.

## High-level architecture

### Storage engine layers

- `mem_table/`:
  - In-memory mutable table implemented as a skip list with arena-style PMR allocation.
  - Records are stored in encoded binary payload form (`internal_key_size + key + seq/type pack + value_size + value`).
- `db/mini_kv_store.cpp` + `db/includes/mini_kv_store.h`:
  - Main engine facade (`get/put/erase`).
  - Write path: WAL append first, then memtable insert.
  - Rotation: when memtable hits threshold, it is moved to immutable queue and flushed by a background thread.

### Persistence and compaction

- WAL: `wal.log` replayed on startup.
- SSTables: `<id>.sst` files generated by flush/compaction.
- Manifest: `MANIFEST` stores `next_sst_id` and per-level SST id layout.
- Read path order:
  1. Active memtable
  2. Immutable memtable queue
  3. Level files (`levels_`) with L0 reverse-time scan and deeper-level indexed selection
- Compaction:
  - Triggered when L0 file count reaches threshold.
  - Merges L0 + L1 with sequence-aware conflict resolution.
  - Drops tombstones during merge output.

### Read acceleration

- Sparse per-SST block index cache (`index_cache_`).
- Per-SST Bloom filter cache (`bloom_cache_`) for negative filtering.
- Block-level LRU cache keyed by `(sst_id, block_offset)` to avoid repeated disk reads.

### Binary/network protocol

- Protocol types are in `server/includes/wire_protocol.h`.
- Fixed 10-byte header (`magic`, `opcode`, `key_len`, `value_len`) plus variable body.
- `stress_test.py` is aligned to this protocol (`<BBII` header layout).

## Concurrency model (engine)

- Foreground reads/writes coordinate via shared/exclusive mutexes around active memtable.
- Flush pipeline uses a background thread and queue guarded by condition variable.
- Manifest/cache structures have separate locks from memtable locks.
- Destructor joins the background compaction thread and flushes WAL cleanly.
